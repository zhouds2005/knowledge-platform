# 部门知识共享平台 — 系统迁移文档

## 概述

将本系统从当前开发机迁移到办公室计算机（Windows + WSL），在 WSL 中继续测试和开发。

**迁移内容：**
- 项目源代码
- PostgreSQL 数据库（结构 + 种子数据）
- 测试套件（集成测试 + E2E 浏览器测试）
- 开发环境配置

---

## 一、目标环境要求

在办公室计算机的 WSL 中安装以下软件：

### 1.1 Node.js（≥ 22）

```bash
# 使用 nvm 安装（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.bashrc
nvm install 22
node --version   # 应显示 v22.x.x
```

### 1.2 Docker（用于 PostgreSQL + Nextcloud）

```bash
# 在 Windows 侧安装 Docker Desktop，WSL 中会自动可用
# 验证：
docker --version
docker compose version
```

### 1.3 Nextcloud（可选，网盘功能需要）

```bash
# 方式1：Docker（推荐）
docker run -d --name nextcloud -p 8080:80 nextcloud

# 方式2：snap（WSL Ubuntu）
sudo snap install nextcloud
sudo nextcloud.manual-install admin admin123
```

## 二、迁移步骤

### 步骤 1：复制项目文件

将打包文件 `knowledge-platform.zip` 复制到办公室计算机，在 WSL 中解压：

```bash
# 假设文件放在 Windows 桌面，WSL 中路径为 /mnt/c/Users/你的用户名/Desktop/
unzip /mnt/c/Users/你的用户名/Desktop/knowledge-platform.zip -d ~/
cd ~/knowledge-platform
```

### 步骤 2：安装依赖

```bash
npm install
npx playwright install chromium
```

### 步驟 3：配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，修改以下内容：

```env
# PostgreSQL — 如果使用 docker compose，保持不变
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/knowledge_platform

# 修改为随机字符串
SESSION_SECRET=your-random-secret-here

# Nextcloud — 按你实际部署的地址填
NEXTCLOUD_URL=http://localhost:8080
NEXTCLOUD_USER=admin
NEXTCLOUD_PASS=admin123
```

### 步骤 4：启动数据库

```bash
# 如果项目有 docker-compose.yml
docker compose up -d db

# 等 PostgreSQL 启动
sleep 5
```

### 步骤 5：创建数据库 + 导入数据

```bash
# 创建数据库（如果不存在）
docker compose exec db psql -U postgres -c "CREATE DATABASE knowledge_platform;" 2>/dev/null || true

# 推送表结构
npx drizzle-kit push --force

# 导入种子数据
bash scripts/restore-db.sh
```

如果 `restore-db.sh` 报错，可以手动执行：
```bash
node -e "
const pg = require('pg');
const fs = require('fs');
require('dotenv/config');
// ... 见 scripts/restore-db.sh 中的完整脚本
"
```

### 步骤 6：验证

```bash
# 启动服务
npm run server &    # 后端 → http://localhost:3002
npm run dev &       # 前端 → http://localhost:5173

# 运行全部测试
npm test            # 43 个集成测试，约 5 秒
npm run test:e2e    # 17 个浏览器测试，约 25 秒
```

打开浏览器访问 `http://localhost:5173`，用以下账号登录：

| 邮箱 | 密码 | 角色 |
|------|------|------|
| admin@company.com | （种子数据中的密码） | 管理员 |
| test@test.com | test123 | 编辑者 |

## 五、常见问题

### Q: `drizzle-kit push` 报 "Interactive prompts require a TTY"

```bash
npx drizzle-kit push --force
```

### Q: 测试报数据库连接失败

确认 PostgreSQL 在运行：
```bash
docker compose ps
# 或
sudo service postgresql status
```

确认 `.env` 中 `DATABASE_URL` 正确：
```bash
grep DATABASE_URL .env
```

### Q: Playwright 测试全部超时

检查 Vite 是否绑定到 0.0.0.0：
```bash
# package.json 中 dev 命令应为：
"dev": "vite --host 0.0.0.0"
```

### Q: Nextcloud 未安装，网盘功能不能用

网盘功能依赖 Nextcloud WebDAV。如果暂时不需要，可以跳过——其他功能不受影响。
