# 部署检查清单

## 首次部署（Ubuntu 24.04）

```bash
# 1. 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash -
sudo apt install -y nodejs

# 2. 装 pm2（进程守护）
npm install -g pm2

# 3. 装依赖
cd /path/to/knowledge-platform
npm install

# 4. 装 Playwright 浏览器
npx playwright install chromium --with-deps

# 5. 启动 PostgreSQL（已有 docker-compose）
docker compose up -d db

# 6. 建表 + 灌数据
npx drizzle-kit push
npx tsx server/db/seed.ts

# 7. 配置 .env
cp .env.example .env
# 编辑 .env，改 SESSION_SECRET 为随机字符串

# 8. 部署
bash scripts/deploy.sh

# 9. 设开机自启
pm2 startup
pm2 save
```

## 每次部署前

- [ ] `npx vitest run` — 78 个测试全绿
- [ ] `npx playwright test` — 57 个 E2E 全绿
- [ ] `git status` — 没有未提交的改动
- [ ] 数据库正常运行（`pg_isready -U postgres -h localhost`）
- [ ] `npm run build` 能成功

## 部署

```bash
bash scripts/deploy.sh
```

## 部署后验证

- [ ] 打开 http://IP:5173 → 能看到登录页
- [ ] 用 test@test.com / test123 登录 → 能进首页
- [ ] `curl http://localhost:3002/api/health` → `{"status":"ok"}`
- [ ] `pm2 status` → knowledge-api 和 knowledge-web 都是 `online`
- [ ] `pm2 logs --lines 20` → 没有红色报错

## 出问题怎么办

| 现象 | 操作 |
|------|------|
| 页面打不开 | `pm2 logs knowledge-web` 看最后 20 行 |
| 接口报错 | `pm2 logs knowledge-api` 查找 Error |
| 数据不对 | `bash scripts/backup.sh` 先备份再排查 |
| 部署后彻底挂了 | `bash scripts/rollback.sh` 一键回滚 |
