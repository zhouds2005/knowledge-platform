# 统一知识对象模型 + RBAC + 网盘集成 — 设计方案

日期：2026-06-12

## 1. 目标

将文档管理、Wiki、Nextcloud 网盘通过统一知识对象模型整合为"内部知识共享体系"，
叠加文档级 RBAC、空间审核流、跨模块搜索推荐。

## 2. 系统架构

```
                        统一知识门户
                 (搜索入口、推荐Feed、空间首页)

  文档模块          Wiki 模块          网盘模块 (Nextcloud)
  上传/归档         创建/协编           浏览/上传/同步
  版本对比          协作编辑            文件挂载到对象
  借阅审批

                    统一知识对象层 (KnowledgeObject)
          类型映射 | 权限 | 状态机 | 全文搜索 | 标签 | 统计

      RBAC 引擎       审核流引擎       通知引擎       知识图谱

        部门 (Department)         知识空间 (KnowledgeSpace)

                  PostgreSQL + Nextcloud API
```

## 3. 核心数据模型

### 3.1 KnowledgeObject（统一知识对象）

```yaml
KnowledgeObject:
  id:              UUID (PK)
  type:            enum("document" | "wiki" | "drive_file")
  title:           string
  description:     text (nullable)
  tags:            string[] (nullable)
  department_id:   FK → Department
  space_id:        FK → KnowledgeSpace

  # 状态与审核
  status:          enum("draft" | "pending_review" | "published" | "archived")
  reviewer_id:     FK → User (nullable)
  published_at:    datetime (nullable)
  version:         int (default 1)

  # 权限控制
  visibility:      enum("space" | "department" | "public")
  owner_id:        FK → User
  editor_ids:      FK[] → User (nullable)
  viewer_ids:      FK[] → User (nullable)

  # 来源多态关联
  source_table:    string
  source_id:       UUID

  # 全文搜索
  search_vector:   tsvector

  # 统计
  view_count:      int (default 0)
  created_at:      datetime
  updated_at:      datetime
```

### 3.2 Department（部门）

```yaml
Department:
  id:              UUID (PK)
  name:            string
  parent_id:       FK → Department (nullable, 支持多层)
```

### 3.3 KnowledgeSpace（知识空间）

```yaml
KnowledgeSpace:
  id:                  UUID (PK)
  department_id:       FK → Department
  name:                string
  description:         text (nullable)
  default_reviewer_id: FK → User (nullable)
  auto_publish:        bool (default false)
```

### 3.4 ObjectPermission（文档级例外权限）

```yaml
ObjectPermission:
  id:            UUID (PK)
  object_id:     FK → KnowledgeObject
  grantee_type:  enum("user" | "role")
  grantee_id:    UUID | string
  permission:    enum("view" | "edit" | "review")
```

### 3.5 ReviewRecord（审核记录）

```yaml
ReviewRecord:
  id:          UUID (PK)
  object_id:   FK → KnowledgeObject
  reviewer_id: FK → User
  action:      enum("approved" | "rejected")
  comment:     text (nullable, 驳回时必填)
  created_at:  datetime
```

## 4. RBAC 授权模型

### 4.1 角色层（全局）

- **admin**：全部模块、全部对象、全部操作
- **editor**：可创建/编辑文档和 Wiki，可上传网盘文件
- **viewer**：只读浏览和搜索

### 4.2 空间层

- **space_admin**：空间审核人，可审批/驳回该空间内的发布请求
- **space_editor**：该空间内可编辑
- **space_viewer**：限制仅对该空间可见

### 4.3 文档层

- **owner**：创建者，始终可编辑自己的草稿
- **editor_ids**：额外可编辑者
- **viewer_ids**：额外可见者（跨部门共享例外）

### 4.4 权限判定流程

```
1. 用户是 admin？                  → 允许
2. 对象 status ≠ "published"？    → 检查是否 owner 或 reviewer（否则拒绝）
3. 对象 visibility = "public"？    → 允许查阅
4. 用户与对象同 department？       → 允许查阅 + 检查 editor_ids
5. 用户在对象的 viewer_ids 里？    → 允许查阅
6. 以上都不满足                    → 拒绝
```

## 5. 空间审核流

### 5.1 状态机

```
draft → pending_review → published
  ↑                           ↓
  └────── rejected ←──────────┘
                               ↓
                           archived
```

### 5.2 角色操作矩阵

| | draft | pending_review | published | archived |
|---|---|---|---|---|
| owner | 编辑、提交审核、删除 | 撤回、查看 | 查看 | 查看 |
| space_editor | 协编 | 查看 | 编辑（生成新草稿） | 查看 |
| reviewer | — | 通过/驳回 | — | — |
| space_viewer | 不可见 | 不可见 | 阅读 | — |
| admin | 全部 | 全部 | 全部 | 恢复/删除 |

### 5.3 版本策略

- 编辑已发布对象 → 自动创建新草稿版本（原发布版本不受影响）
- 草稿审核通过 → 草稿替换为当前发布版本，旧版本存快照
- 驳回 → 草稿保留，reviewer 附驳回理由，owner 可修改后重新提交
- 同一时刻最多一个已发布版本 + 一个草稿

### 5.4 通知节点

- 提交审核 → 通知空间 reviewer
- 审核通过 → 通知 owner + 空间内成员
- 驳回 → 通知 owner，附理由
- 新版本发布 → 通知"关注"该对象的用户

## 6. 网盘集成（Nextcloud）

- 网盘文件作为 KnowledgeObject（type = "drive_file"）
- 本系统内浏览 Nextcloud 目录结构
- 本地文件上传 → 自动同步到 Nextcloud
- 文档详情页 / Wiki 页面可挂载引用 Nextcloud 文件作为附件
- 网盘文件纳入统一搜索索引

## 7. 知识门户

- 统一搜索入口：跨文档 / Wiki / 网盘文件名 / 标签全文搜索
- 空间首页：每个 KnowledgeSpace 展示其下最近更新的知识对象
- 推荐 Feed：基于用户 department 和浏览历史推荐相关内容
- 知识图谱：展示知识对象间的引用关系

## 8. 实施清单

### 8.1 新增数据表
- `departments`
- `knowledge_spaces`
- `knowledge_objects`
- `object_permissions`
- `review_records`

### 8.2 存量迁移
- 现有 `documents` 表 + `wiki_pages` 表各增加 `knowledge_object_id` 外键
- 迁移脚本为每条现有记录生成对应 KnowledgeObject

### 8.3 新增 API
- `GET/POST /api/knowledge/search` — 统一搜索
- `GET /api/knowledge/:id` — 知识对象详情
- `POST /api/knowledge/:id/submit-review` — 提交审核
- `POST /api/knowledge/:id/review` — 审核通过/驳回
- `GET /api/knowledge/spaces/:spaceId` — 空间首页聚合
- 权限判定中间件集成

### 8.4 改造页面
- Documents / Wiki / Nextcloud 页面接入权限判定和审核状态机
- 创建/编辑表单加标签、部门/空间选择

### 8.5 新增页面
- 知识门户首页
- 空间管理页
- 审核队列页（reviewer 视角）
